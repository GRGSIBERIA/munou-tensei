# 無能転生 要件定義（ドラフト）

## 1. 目的
- 学校の学年全体が中世風なろう系ヨーロッパに転生し、プレイヤーは神の視点で学年を管理するシミュレーションゲーム。
- 1プレイは短くて20分、長くて数時間程度で、短時間でも進捗が感じられるゲームサイクルを実現する。

## 2. 想定プラットフォーム
- PC（Windows/Mac）を対象とする。

## 3. ターゲットユーザー
- シミュレーションゲームが好きな18～45歳
- Paradoxのゲームを参考にし、硬派で難易度と複雑性の高いゲームを指向する

## 4. コアゲームループ
1. リアルタイムに進行する
2. 生徒に職業を割り振る
3. 成功/失敗に応じてステータス・資源が変動する
4. 生徒を観察しながら職業の割り振りを継続するか取り下げるか考える

## 5. 機能要件（MVP）
- キャラクター基本ステータス（例: 体力、知力、器用さ）。
- 行動選択UI（最低3種類の行動）。
- 成否判定ロジック（確率 + ステータス補正）。
- 報酬システム（熟練度、通貨のいずれか）。
- セーブ/ロード（ローカル保存）。
- チュートリアル（初回プレイ3分以内で基本操作を理解可能）。

## 6. 非機能要件（MVP）
- 起動時間: 10秒以内（ターゲットPC環境）。
- 平均フレームレート: 60fps（1080p、標準設定）。
- クラッシュ率: 1%未満（内部テスト）。
- 主要操作はキーボード/ゲームパッド双方で実行可能。

## 7. 受入基準（初期）
- 新規プレイヤーが10分以内に特定の生徒の「1回の成長サイクル」を完了できる。
- 連続5回の行動結果に対して、成否判定が仕様どおり再現される。
- セーブ後に再起動しても、ステータスと進行状況が復元される。

## 8. 今後の検討項目
- 世界観・シナリオの具体化。
- 育成ビルドの多様性（スキルツリー等）。
- リプレイ性向上のためのランダムイベント設計。
